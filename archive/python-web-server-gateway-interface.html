Python Web Server Gateway Interface

Phillip Eby (the PEAK guy) rewrote the WSGI interface spec. It describes a standard interface between servers and applications. Most commonly that would actually be between servers and frameworks. Discussion is on Web-SIG. I think the proposal looks very straight-forward and general, and it could be quite useful in straightening out the mess we're in with respect to Python web programming.
