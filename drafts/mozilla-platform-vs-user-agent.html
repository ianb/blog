Mozilla’s engineering conflict: platform vs user agent

Mozilla’s conflicted choice: between platform and user agent

Mozilla has always had a hard time understanding its direction. I think that’s part of being a mission-led organization, there’s not ultimately a single and clear metric of success. It’s not quite profit, not quite impact, and there’s a dozen kinds of impact we think about.

All considered I don’t think Mozilla is particularly political. I don’t think “political” is itself a terrible thing, it’s just how we describe decision processes when there’s no overriding rule or clear criteria that applies the same to two potential options. For instance, Mozilla has no clear way to decide when to do something to increase privacy that decreases web compatibility. If we always choose privacy then people will stop using Firefox (and yes, we know people will stop using it) and we can’t deliver the privacy features to as many people.
