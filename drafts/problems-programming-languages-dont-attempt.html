Problems Programming Languages Don’t Attempt To Solve

What’s in a new programming language?

I for one am a bit bored, it seems like the same problems are addressed over and over in different ways. Types, generics, concurrency, side effects.

It feels like we’re stuck.

Here’s some things that I see as concerns that programmers have, the things that actually take the time and effort, that maybe a programming language could attempt to address:

Problems

Repeatability

Understanding change and effect of change

Testing, mocks, and environments

Persistence and systems evolving at different paces

Understanding data flow

Dynamic scope

Source control organization

Self-documentation and the maintenance of that documentation

Something to do with the management of hybrid systems

Global introspection / process introspection

Can you find

For reasons I don’t understand, this is something that exists but seems unused. For instance, in Python there is gc.get_objects() or gc.get_referrers().

We watch lines of code but we don’t often watch objects. (Functional techniques even get in the way of this due to the replaceability of objects!)

Wholelistic thinking

For any of these, I’m sure you are thinking: well, language X solves that with Y, language Z has another approach, there’s a programming pattern, and so on.

And that’s true. We do manage to keep successfully writing code and solving these problems. And before structured programming people still wrote code that had structure, it just wasn’t embodied in the programming language.

That said, I think there’s a possibility that by looking at the entire environment, language, tooling, and programming UI, there may be something new to find.

What’s language, what’s environment?
