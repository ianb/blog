Why do you want to be popular?

Mozilla’s undergoing a lot of changes in the last few years. A huge part of this is related to the question of choosing why Mozilla does what it does.

For a while I would go around asking people at Mozilla how they would tell someone else why should you use Firefox? It’s a frustratingly hard question for us to answer, and answers often rested on the values of Mozilla… and sadly not how those values were embodied in Firefox itself. But there was a subtly different kind of answer that I also got: people should use Firefox because its popularity is what gives Mozilla the influence to protect the web. In some sense this is a more honest answer, but it also says nothing about why you should use Firefox. And so it went for Mozilla and Firefox…

Things are changing in Mozilla! The Firefox Group no longer accepts these kinds of answers. But that still doesn’t actually answer the question: why should you use Firefox? There is a tension here, unresolved questions.

In an effort to resolve these questions we are trying very hard to focus our planning on user value. This is different from focusing on, say, open standards, privacy-for-privacy, or acting as an example of good corporate behavior. Mozilla is a mission-led organization, it does have values that drive our action, but we’re using that more as a filter and less as a driver of our activity. I think this is good: when we were more mission-focused instead of product-focused it made it all too easy for each person to project personal goals onto the organization, with disappointment for everyone involved (myself included).

But user value, the pursuit of popularity… I’m skeptical of this too. Sure, it would be awesome to have a breakout hit… as though we can just will ourselves to achieve this. But we must instead prepare ourselves – expect as an inevitability – that everything we do will be met with disinterest. We’re never going to simply achieve success. And what then? Do we pivot? Do we flail? Do we retreat? Do we figure out what it is we understand how to do, then come up with reasoning that supports a clear path?

So I would assert that we must not start with user value and we must not simply pursue popularity. We have to decide what Mozilla and Firefox organizations desperately want to be popular. Not everything that we want may be achievable, but we still have to start with desire, because it’s desire that will allow us to go through the challenging and sometimes painful process to actually achieve success, to keep working when it’s not going well and none of it feels rewarding.

Context Graph hints at an idea. But it’s fragile, incomplete; if someone said “eh, let’s not do that” all efforts would immediately scatter to the wind.

It’s not Mozilla’s goal, but I will state my goal with Pageshot. It is not to be useful, it’s not just jamming on the popularity of screenshots. In the three-way struggle for power between site owner, user agent, and user, I want to shift power to the user (not even to the user agent, whose role we have kept limited in PageShot). And even though PageShot has kept getting pushed aside for other priorities, even though it has no sponsor, even though I can see many dragons up ahead, we keep going.

Should our products tell a story? Do you want to get on board, see where this is going? Not everyone thinks a user agent should tell a story. Open standards work against this, towards commodification (and towards site owner dominance, given the dynamics of these bodies). Google doesn’t want the user agent to tell a story, except maybe a story of competence and accuracy, the cold depersonalized politics and aesthetic Google brings to the web and embodied in so many of their products. Mozilla has learned far too much from Google. Can Firefox have a voice? This is a question for the institution of Mozilla, and since institutions aren’t people the only way we’ll get an answer is if it does or does not express a voice in its product.
